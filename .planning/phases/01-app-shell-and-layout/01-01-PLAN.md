  <action>
    1. In `nano_claude/app.py`, add an `on_resize` handler (or `watch_size` reactive watcher on the app):
       ```python
       def on_resize(self, event: Resize) -> None:
           """Handle terminal resize -- collapse panels at small sizes."""
           # Defer to after layout recalculation (Pitfall 4 from research)
           self.call_later(self._handle_responsive_collapse, event.size.width)
       ```

    2. Implement `_handle_responsive_collapse(self, terminal_width: int)`:
       - Import `COLLAPSE_TREE_THRESHOLD` and `COLLAPSE_CHAT_THRESHOLD` from `nano_claude.config.settings`
       - If `terminal_width < COLLAPSE_TREE_THRESHOLD`:
           - `self.query_one("#file-tree").add_class("hidden")`
       - Else:
           - `self.query_one("#file-tree").remove_class("hidden")`
       - If `terminal_width < COLLAPSE_CHAT_THRESHOLD`:
           - `self.query_one("#chat").add_class("hidden")`
       - Else:
           - `self.query_one("#chat").remove_class("hidden")`
       - After toggling, call `self._apply_panel_widths()` to update fr values on visible panels

    3. Implement `_apply_panel_widths(self)`:
       ```python
       def _apply_panel_widths(self) -> None:
           panels = {
               "file-tree": self.tree_width,
               "editor": self.editor_width,
               "chat": self.chat_width,
           }
           for panel_id, fr_val in panels.items():
               try:
                   panel = self.query_one(f"#{panel_id}")
                   if not panel.has_class("hidden"):
                       panel.styles.width = f"{fr_val}fr"
               except Exception:
                   pass
       ```

    4. Also call `_handle_responsive_collapse` in `on_mount` so that if the app starts in a small terminal, panels collapse immediately:
       ```python
       def on_mount(self) -> None:
           self._handle_responsive_collapse(self.size.width)
       ```
